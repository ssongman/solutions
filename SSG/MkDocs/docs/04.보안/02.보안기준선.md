# 보안 기준선 (Security Baseline)

Azure 랜딩존의 보안 기준선은 **Microsoft Defender for Cloud**를 중심으로  
모든 구독에 일관된 보안 상태를 유지합니다.

---

## Defender for Cloud 구성

### 구독별 활성화 플랜

| 플랜 | 대상 | 비용 |
|------|------|------|
| Defender for Servers P2 | 모든 VM | $0.02/VM-hour |
| Defender for SQL | Azure SQL, SQL on VM | $0.015/VM-hour |
| Defender for Storage | Storage Account | $10/month per account |
| Defender for Containers | AKS | $7/VM-core-hour |
| Defender for Key Vault | Key Vault | $0.02/1만 트랜잭션 |

```bash
# 구독의 모든 Defender 플랜 활성화
az security pricing create \
  --name VirtualMachines \
  --tier Standard
```

## Microsoft Cloud Security Benchmark (MCSB)

랜딩존 전체에 **MCSB (구 Azure Security Benchmark)** 를 기본 보안 정책으로 적용합니다.

```bash
# 관리 그룹에 MCSB Initiative 할당
az security assessment-metadata create \
  --name "mcsb-assignment" \
  --scope "/providers/Microsoft.Management/managementGroups/root-mg"
```

### 주요 통제 항목

| 영역 | 통제 내용 |
|------|----------|
| 네트워크 보안 | NSG 필수, 불필요 공인 IP 제거 |
| ID 관리 | MFA 강제, 레거시 인증 차단 |
| 권한 있는 접근 | PIM 활성화, Just-in-Time VM 접근 |
| 데이터 보호 | 저장 데이터 암호화, TLS 1.2 이상 |
| 로그 & 모니터링 | 진단 로그 → Log Analytics 전송 |
| 취약점 관리 | Qualys/TVM 통합 취약점 스캔 |

## Just-in-Time (JIT) VM 접근

SSH/RDP 포트를 평소에는 차단하고 **요청 시에만 일시적으로 개방**합니다.

```
평소: NSG에서 22/3389 차단
    │
    │ JIT 요청 (포털/CLI/API)
    │
승인 후 최대 3시간 특정 IP만 허용
    │
시간 만료 → 자동 차단
```

```bash
# JIT 정책 설정
az security jit-policy create \
  --resource-group rg-corp \
  --name default \
  --vm-name vm-app \
  --ports '[{"number": 22, "protocol": "TCP", "allowedSourceAddressPrefix": "*", "maxRequestAccessDuration": "PT3H"}]'
```

## 보안 경보 및 인시던트 대응

### Microsoft Sentinel 연동

```
Defender for Cloud 경보
    │
    ▼
Microsoft Sentinel (SIEM)
    ├── 분석 규칙으로 인시던트 생성
    ├── 자동화 플레이북 실행
    └── Teams / PagerDuty 알림
```

### 주요 탐지 규칙

| 규칙 | 심각도 |
|------|--------|
| 불가능한 이동 감지 (다른 국가 동시 로그인) | High |
| 관리자 계정 무차별 대입 공격 | High |
| 대량 리소스 삭제 | Critical |
| 비정상 아웃바운드 트래픽 | Medium |
| Key Vault 대량 조회 | Medium |

- **NIST SP 800-53**
