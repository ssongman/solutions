# 허브-스포크 네트워크 토폴로지

허브-스포크(Hub-Spoke)는 Azure 랜딩존의 **표준 네트워크 토폴로지**입니다.  
중앙 허브 VNet을 통해 모든 트래픽을 제어하고, 각 워크로드는 스포크 VNet에 격리됩니다.

---

## 아키텍처 개요

```
온프레미스
    │
    │  ExpressRoute / VPN
    │
[Hub VNet - Connectivity 구독]
    ├── Azure Firewall          ← 모든 트래픽 검사
    ├── ExpressRoute Gateway    ← 온프레미스 연결
    ├── VPN Gateway             ← 백업 연결
    ├── Azure Bastion           ← 안전한 VM 접속
    └── Private DNS Zone        ← 내부 DNS 해석
         │
         ├── VNet Peering ──── [Spoke VNet 1 - Corp]
         │                          └── App VM, DB
         │
         └── VNet Peering ──── [Spoke VNet 2 - Online]
                                    └── Web App, API
```

---

## Hub VNet 구성

### 권장 주소 공간

| 서브넷 | CIDR | 용도 |
|--------|------|------|
| AzureFirewallSubnet | 10.0.0.0/26 | Azure Firewall (필수 이름) |
| GatewaySubnet | 10.0.0.64/27 | ExpressRoute/VPN GW (필수 이름) |
| AzureBastionSubnet | 10.0.0.96/27 | Azure Bastion (필수 이름) |
| ManagementSubnet | 10.0.0.128/28 | 관리 VM, Jump Server |

### Azure Firewall 역할
- 스포크 → 인터넷 아웃바운드 제어
- 스포크 ↔ 스포크 East-West 트래픽 검사
- 온프레미스 ↔ 스포크 트래픽 검사

## Spoke VNet 구성

### Corp Spoke (사내망 연결)
```
Spoke VNet CIDR: 10.1.0.0/16
├── AppSubnet      10.1.1.0/24   (Application VM)
├── DBSubnet       10.1.2.0/24   (Database)
└── PrivateEPSubnet 10.1.3.0/24  (Private Endpoint)
```

- Hub VNet과 **VNet Peering** 연결
- 인터넷 아웃바운드 → Hub의 Azure Firewall 경유
- 공인 IP 생성 **Deny** (Azure Policy 적용)

### Online Spoke (인터넷 서비스)
```
Spoke VNet CIDR: 10.2.0.0/16
├── FrontendSubnet  10.2.1.0/24  (App Gateway, WAF)
├── AppSubnet       10.2.2.0/24  (App Service, AKS)
└── DBSubnet        10.2.3.0/24  (Database)
```

## VNet Peering 설정

```bash
# Hub → Corp Spoke Peering
az network vnet peering create \
  --name hub-to-corp \
  --resource-group rg-connectivity \
  --vnet-name vnet-hub \
  --remote-vnet vnet-corp \
  --allow-forwarded-traffic true \
  --allow-gateway-transit true

# Corp Spoke → Hub Peering
az network vnet peering create \
  --name corp-to-hub \
  --resource-group rg-corp \
  --vnet-name vnet-corp \
  --remote-vnet vnet-hub \
  --allow-forwarded-traffic true \
  --use-remote-gateways true
```

## 라우팅 (UDR)

각 스포크의 서브넷에 **User Defined Route(UDR)** 을 설정하여 모든 트래픽이 Azure Firewall을 경유하도록 합니다.

```bash
# 기본 라우트 → Azure Firewall로 강제
az network route-table route create \
  --name default-to-firewall \
  --route-table-name rt-corp-app \
  --address-prefix 0.0.0.0/0 \
  --next-hop-type VirtualAppliance \
  --next-hop-ip-address 10.0.0.4  # Azure Firewall Private IP
```
