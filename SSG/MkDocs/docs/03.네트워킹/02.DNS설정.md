# DNS 설계 (Private DNS Zone)

Azure 랜딩존에서 DNS는 **Private Endpoint와 내부 서비스 이름 해석**을 위한 핵심 인프라입니다.  
Hub VNet에 중앙 집중식 Private DNS Zone을 구성하고 모든 스포크가 공유합니다.

---

## DNS 아키텍처

```
[VM in Spoke VNet]
    │
    │  storage.privatelink.blob.core.windows.net 질의
    │
[Azure-provided DNS: 168.63.129.16]
    │
    │  Private DNS Zone 링크 (Hub VNet)
    │
[Private DNS Zone: privatelink.blob.core.windows.net]
    │
    └── storageaccount.blob.core.windows.net → 10.0.3.5 (Private IP)
```

---

## Private DNS Zone 목록 (주요 PaaS 서비스)

| 서비스 | Private DNS Zone |
|--------|-----------------|
| Blob Storage | privatelink.blob.core.windows.net |
| SQL Database | privatelink.database.windows.net |
| Key Vault | privatelink.vaultcore.azure.net |
| ACR | privatelink.azurecr.io |
| AKS API Server | privatelink.koreacentral.azmk8s.io |
| Service Bus | privatelink.servicebus.windows.net |
| App Service | privatelink.azurewebsites.net |

## 구성 절차

### 1. Private DNS Zone 생성 (Hub 구독)

```bash
az network private-dns zone create \
  --resource-group rg-connectivity \
  --name "privatelink.blob.core.windows.net"
```

### 2. Hub VNet에 DNS Zone 링크

```bash
az network private-dns link vnet create \
  --resource-group rg-connectivity \
  --zone-name "privatelink.blob.core.windows.net" \
  --name link-hub \
  --virtual-network vnet-hub \
  --registration-enabled false
```

### 3. Spoke VNet에도 동일하게 링크

```bash
# Corp Spoke 연결
az network private-dns link vnet create \
  --resource-group rg-connectivity \
  --zone-name "privatelink.blob.core.windows.net" \
  --name link-corp \
  --virtual-network vnet-corp \
  --registration-enabled false
```

### 4. Private Endpoint 생성 시 DNS 레코드 자동 등록

```bash
az network private-endpoint create \
  --name pe-storage \
  --resource-group rg-corp \
  --vnet-name vnet-corp \
  --subnet PrivateEPSubnet \
  --private-connection-resource-id "/subscriptions/.../storageAccounts/mystorage" \
  --group-id blob \
  --connection-name conn-storage

# DNS 레코드 자동 등록 설정
az network private-endpoint dns-zone-group create \
  --resource-group rg-corp \
  --endpoint-name pe-storage \
  --name zonegroup \
  --private-dns-zone "privatelink.blob.core.windows.net" \
  --zone-name blob
```

## 온프레미스 DNS 연동

온프레미스 서버가 Azure Private DNS를 해석해야 하는 경우 **Azure DNS Private Resolver** 를 사용합니다.

```
온프레미스 DNS 서버
    │  privatelink.blob.core.windows.net 조건부 전달
    │
Azure DNS Private Resolver (Hub VNet)
    │  Inbound Endpoint IP: 10.0.0.200
    │
Private DNS Zone 해석
```

```bash
# DNS Private Resolver 생성
az dns-resolver create \
  --name dns-resolver-hub \
  --resource-group rg-connectivity \
  --location koreacentral \
  --id "/subscriptions/.../virtualNetworks/vnet-hub"
```
