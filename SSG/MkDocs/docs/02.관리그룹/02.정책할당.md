# 정책 할당 (Azure Policy)

Azure Policy는 **리소스 배포 시 자동으로 규칙을 평가하고 적용**하는 거버넌스 도구입니다.  
랜딩존에서는 정책을 관리 그룹 단위로 할당하여 모든 구독에 일관성 있게 적용합니다.

---

## 정책 효과 (Effect) 종류

| Effect | 동작 | 사용 시점 |
|--------|------|----------|
| **Deny** | 비준수 리소스 배포 차단 | 허용 불가 설정 강제 |
| **Audit** | 배포는 허용, 비준수 리소스 기록 | 모니터링 목적 |
| **Append** | 리소스에 속성 자동 추가 | 태그 자동 추가 |
| **DeployIfNotExists** | 리소스 배포 시 연관 리소스 자동 생성 | 진단 설정, 에이전트 설치 |
| **Modify** | 기존 리소스 속성 수정 | 태그 정규화 |

---

## 랜딩존 필수 정책 목록

### Root Management Group (전사 공통)

```
✅ 허용 리전 제한 (Korea Central, Korea South만 허용)
✅ 필수 태그 강제 (Environment, Owner, CostCenter)
✅ 허용 리소스 유형 제한
✅ Azure Security Benchmark 적용
✅ 진단 로그 → Log Analytics 자동 전송 (DeployIfNotExists)
```

### Platform / Connectivity

```
✅ VNet에 Network Watcher 자동 활성화
✅ NSG Flow Log 강제 활성화
✅ Private Endpoint만 허용 (PaaS 서비스)
```

### Landing Zones / Corp

```
✅ VM에 Defender for Endpoint 자동 설치
✅ 공인 IP 생성 Deny
✅ Azure Firewall을 통한 인터넷 아웃바운드만 허용
```

### Sandbox

```
✅ VM SKU 제한 (D2s_v3 이하만 허용)
✅ 월 예산 초과 시 알림
```

## 정책 할당 예시 (Azure CLI)

```bash
# 관리 그룹에 허용 리전 제한 정책 할당
az policy assignment create \
  --name "allowed-locations" \
  --scope "/providers/Microsoft.Management/managementGroups/root-mg" \
  --policy "e56962a6-4747-49cd-b67b-bf8b01975c4f" \
  --params '{"listOfAllowedLocations": {"value": ["koreacentral", "koreasouth"]}}'
```

---

## Policy Initiative (정책 묶음)

여러 정책을 하나의 **Initiative(이니셔티브)** 로 묶어 관리합니다.

```
Landing Zone Baseline Initiative
├── 허용 리전 제한
├── 필수 태그 강제
├── 진단 로그 설정
└── Defender 에이전트 자동 설치
```

```bash
# Initiative 정의 후 관리 그룹에 할당
az policy set-definition create \
  --name "lz-baseline" \
  --definitions "@lz-baseline-policies.json"
```

---

## 비준수 리소스 처리

정책 할당 후 기존 비준수 리소스는 **Remediation Task**로 일괄 수정합니다.

```bash
az policy remediation create \
  --name "fix-diagnostic-settings" \
  --policy-assignment "lz-diag-settings" \
  --resource-discovery-mode ReEvaluateCompliance
```
