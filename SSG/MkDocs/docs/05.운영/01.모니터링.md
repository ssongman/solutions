# 모니터링 & 운영

Azure 랜딩존에서 모니터링은 **Log Analytics Workspace를 중심으로**  
모든 구독의 로그, 메트릭, 보안 이벤트를 중앙에서 수집합니다.

---

## 모니터링 아키텍처

```
[모든 구독의 리소스]
    │  진단 설정 (DeployIfNotExists 정책으로 자동)
    │
    ▼
[Log Analytics Workspace]     ← Management 구독
    │  (중앙 집중식)
    │
    ├── Azure Monitor Dashboards   ← 운영 대시보드
    ├── Azure Workbooks            ← 커스텀 리포트
    ├── Alert Rules                ← 임계값 경보
    └── Microsoft Sentinel         ← 보안 분석
```

---

## Log Analytics Workspace 구성

### Workspace 배포 (Management 구독)

```bash
az monitor log-analytics workspace create \
  --resource-group rg-management \
  --workspace-name law-central \
  --location koreacentral \
  --sku PerGB2018 \
  --retention-time 90
```

### 데이터 수집 항목

| 데이터 소스 | 테이블 | 용도 |
|------------|--------|------|
| Azure Activity Log | AzureActivity | 리소스 변경 감사 |
| VM 성능 | Perf | CPU, 메모리, 디스크 |
| VM 이벤트 | Event (Windows) / Syslog (Linux) | OS 이벤트 |
| NSG Flow Log | AzureNetworkAnalytics_CL | 네트워크 트래픽 |
| Azure AD 로그인 | SigninLogs | 인증 감사 |
| Defender 경보 | SecurityAlert | 보안 위협 |

## 진단 설정 자동화 (Policy)

[정책 할당](../02.관리그룹/02.정책할당.md) 에서 설정한 **DeployIfNotExists 정책**이  
리소스 배포 시 자동으로 진단 설정을 Log Analytics로 전송합니다.

```bash
# VM 진단 설정 수동 확인
az monitor diagnostic-settings list \
  --resource "/subscriptions/.../virtualMachines/vm-app"
```

## Azure Monitor 경보 규칙

### 인프라 경보

```bash
# CPU 80% 초과 시 경보
az monitor metrics alert create \
  --name "high-cpu-alert" \
  --resource-group rg-management \
  --scopes "/subscriptions/.../virtualMachines/vm-app" \
  --condition "avg Percentage CPU > 80" \
  --window-size 5m \
  --evaluation-frequency 1m \
  --action /subscriptions/.../actionGroups/ops-team
```

### 권장 경보 목록

| 경보 | 임계값 | 심각도 |
|------|--------|--------|
| VM CPU | > 80% (5분) | Warning |
| VM 메모리 | > 85% (5분) | Warning |
| 디스크 여유 공간 | < 10% | Critical |
| Azure Firewall 차단 | > 100건/분 | Warning |
| 구독 예산 | > 80% | Warning |

## KQL 쿼리 예시

### 최근 리소스 삭제 이력

```kusto
AzureActivity
| where OperationNameValue endswith "DELETE"
| where ActivityStatusValue == "Success"
| project TimeGenerated, Caller, ResourceGroup, Resource, OperationNameValue
| order by TimeGenerated desc
| take 50
```

### 실패한 로그인 Top 10

```kusto
SigninLogs
| where ResultType != 0
| summarize FailCount = count() by UserPrincipalName
| order by FailCount desc
| take 10
```

### VM 평균 CPU Top 5

```kusto
Perf
| where ObjectName == "Processor" and CounterName == "% Processor Time"
| where TimeGenerated > ago(1h)
| summarize AvgCPU = avg(CounterValue) by Computer
| order by AvgCPU desc
| take 5
```

## Azure Workbooks 대시보드

Management 구독에 아래 Workbook 템플릿을 배포합니다.

| 대시보드 | 내용 |
|---------|------|
| Landing Zone Overview | 전체 구독 리소스 현황 |
| Security Posture | Defender 점수, 비준수 리소스 |
| Network Traffic | Firewall 로그, NSG Flow |
| Cost Analysis | 구독별 비용 트렌드 |
