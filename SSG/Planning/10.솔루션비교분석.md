# Static Site Generator 비교 분석

## 요구사항 요약

| 항목 | 요건 |
|------|------|
| 배포 타겟 | Azure Static Web Apps, AKS Pod (nginx) |
| CI/CD | GitHub Actions Workflow |
| Markdown 상대경로 링크 | 하위 문서 간 링크 정확히 변환 |
| 입력 | GitHub 저장소의 Markdown 파일 |
| 출력 | 정적 파일 (HTML/CSS/JS) |

# 1. Jekyll

## 개요
- Ruby 기반 / GitHub Pages 공식 지원 SSG
- 2008년부터 사용된 가장 오래된 SSG

## 장점

| 항목 | 내용 |
|------|------|
| GitHub 친화성 | GitHub Pages 기본 엔진, 공식 Actions 제공 |
| 성숙도 | 레퍼런스 및 플러그인 생태계가 가장 풍부 |
| 진입장벽 | `_config.yml` 하나로 시작 가능 |
| Azure 배포 | Static Web Apps 공식 지원 빌드 프리셋 존재 |

## 단점

| 항목 | 내용 |
|------|------|
| **상대경로 링크** | `.md` 링크를 `.html`로 자동 변환하지 않음. `jekyll-relative-links` 플러그인 필요하며 중첩 폴더 구조에서 불안정 |
| 빌드 속도 | Ruby 기반으로 문서 수가 늘수록 급격히 느려짐 |
| 런타임 의존성 | Ruby + Bundler 환경 구성 필요 |
| 네비게이션 | 폴더 구조 → 사이드바 자동 생성 없음, 직접 구성 필요 |
| Docker 이미지 | Ruby 이미지 크기 큼 (~800MB) |

## GitHub Actions 예시

```yaml
- uses: actions/jekyll-build-pages@v1
- uses: actions/upload-pages-artifact@v1
```

## 상대경로 처리 방식
```
# 플러그인 필요
# Gemfile
gem "jekyll-relative-links"

# _config.yml
plugins:
  - jekyll-relative-links
```

**결론**: 상대경로 링크 처리가 불안정하고 빌드 속도 문제로 현재 요건에는 부적합

# 2. Astro

## 개요
- Node.js 기반 / 2022년 등장한 차세대 SSG
- "Islands Architecture" — 필요한 곳만 JS 로드
- React, Vue, Svelte 등 모든 UI 프레임워크와 호환

## 장점

| 항목 | 내용 |
|------|------|
| **상대경로 링크** | `@astrojs/markdown-remark` 가 `.md` → `.html` 자동 변환, 중첩 폴더 완벽 지원 |
| 빌드 결과물 | 순수 정적 HTML/CSS/JS, nginx 서빙에 최적 |
| 유연성 | 폴더 구조 기반 자동 라우팅 (`src/pages/` 하위 파일 = URL) |
| Azure 배포 | Static Web Apps 공식 빌드 프리셋 존재 (`app_build_command: npm run build`) |
| Docker/AKS | `dist/` 폴더를 nginx로 서빙, 이미지 경량 (~20MB) |
| 커스터마이징 | Tailwind CSS, 검색, 다크모드 등 자유롭게 구성 가능 |
| 성능 | Vite 기반 빌드, 매우 빠름 |

## 단점

| 항목 | 내용 |
|------|------|
| 학습곡선 | `.astro` 파일 문법 학습 필요 |
| 문서 특화 기능 | 사이드바 자동 생성 등은 `Starlight` 테마 사용 필요 |
| 설정량 | 초기 세팅이 MkDocs보다 많음 |

## GitHub Actions 예시

```yaml
- run: npm ci
- run: npm run build        # dist/ 생성
- uses: azure/static-web-apps-deploy@v1
  # 또는 docker build → AKS 배포
```

## 상대경로 처리 방식
```
# astro.config.mjs
export default defineConfig({
  markdown: {
    remarkPlugins: [],  // 기본적으로 상대경로 .md → HTML 변환됨
  }
})
```

**결론**: 유연성과 상대경로 처리 모두 우수. 커스텀 레이아웃이 필요하면 최적 선택

# 3. MkDocs (+ Material Theme)

## 개요
- Python 기반 / 기술 문서 특화 SSG
- Material for MkDocs 테마가 사실상 표준

## 장점

| 항목 | 내용 |
|------|------|
| **상대경로 링크** | `.md` 파일 간 상대경로 링크를 완벽하게 `.html`로 변환 |
| 폴더 → 네비게이션 | `nav:` 설정 or 자동 탐색으로 폴더 구조 그대로 사이드바 생성 |
| 문서 특화 | 검색, 태그, 코드 하이라이팅, 다크모드, 버전 관리 기본 내장 |
| 설정 단순 | `mkdocs.yml` 하나로 전체 구성 |
| 현재 구조 적합 | 기존 폴더 계층 구조 그대로 활용 가능 |

## 단점

| 항목 | 내용 |
|------|------|
| Python 의존성 | Python 3.x + pip 환경 필요 |
| 커스터마이징 | Material 테마 범위 내에서만 쉽게 수정 가능 |
| UI 유연성 | React/Vue 컴포넌트 삽입 불가, 완전 커스텀 레이아웃 어려움 |
| Azure 프리셋 | Static Web Apps 공식 프리셋 없음 (custom 빌드 설정 필요) |

## GitHub Actions 예시

```yaml
- run: pip install mkdocs-material
- run: mkdocs build       # site/ 생성
- uses: azure/static-web-apps-deploy@v1
  with:
    app_location: "site"
    skip_app_build: true
```

## 상대경로 처리 방식
```yaml
# mkdocs.yml — 기본 지원, 별도 설정 불필요
site_name: Solutions
theme:
  name: material
```

**결론**: 상대경로 및 문서 구조 자동화가 가장 쉬움. 현재 폴더 구조에 가장 빠르게 적용 가능

# 4. VitePress

## 개요
- Node.js + Vue 기반 / Vite 빌드
- Vue.js 공식 문서에 사용되는 SSG

## 장점

| 항목 | 내용 |
|------|------|
| **상대경로 링크** | `.md` → `.html` 자동 변환, 앵커(`#`) 링크도 정확히 처리 |
| 빌드 속도 | Vite 기반으로 매우 빠름 |
| 기술 문서 특화 | 사이드바, 검색, 코드블록, 다크모드 기본 제공 |
| Vue 컴포넌트 | Markdown 내 Vue 컴포넌트 삽입 가능 |
| Azure/AKS | 빌드 결과(`dist/`) 정적 파일로 nginx 서빙 최적 |

## 단점

| 항목 | 내용 |
|------|------|
| 사이드바 자동 생성 | 폴더 구조 → 사이드바 **자동 변환 없음**, `config.ts`에 수동 정의 or 플러그인 사용 필요 |
| Vue 의존성 | Vue 생태계에 종속 |
| 버전 안정성 | 아직 1.x 초반, 일부 API 변경 가능성 |
| 설정 복잡도 | 문서가 많아질수록 `config.ts` 관리 부담 |

## GitHub Actions 예시

```yaml
- run: npm ci
- run: npm run docs:build   # .vitepress/dist/ 생성
- uses: azure/static-web-apps-deploy@v1
```

## 상대경로 처리 방식
```ts
// .vitepress/config.ts — 기본 지원
export default {
  // 별도 설정 없이 .md 상대경로 링크 자동 변환
}
```

**결론**: 빠르고 깔끔하나 사이드바 자동 생성이 없어 폴더 구조가 많은 경우 관리 부담

# 6. 배포 타겟별 적합도

## Azure Static Web Apps

```
Astro    ✅  공식 빌드 프리셋, oryx 빌더 지원
VitePress ✅  Node.js 빌드 후 dist/ 배포
MkDocs   △   skip_app_build: true 로 우회 가능
Jekyll   ✅  공식 프리셋 있으나 상대경로 불안정
```

## AKS Pod (nginx)

```
모든 SSG 동일: 빌드 결과물(HTML/CSS/JS)을 
nginx:alpine 이미지에 COPY 후 서빙

FROM nginx:alpine
COPY dist/ /usr/share/nginx/html/   # Astro, VitePress
COPY site/ /usr/share/nginx/html/   # MkDocs
```

# 7. 최종 추천

## 1순위: **MkDocs + Material**
> 현재 폴더 계층 구조(폴더 = 카테고리, .md = 문서)에 가장 빠르게 적용 가능  
> 상대경로 링크 변환 자동, 사이드바 자동 생성, 설정 최소화

## 2순위: **Astro (Starlight 테마)**
> 커스텀 레이아웃/디자인이 중요하거나, 향후 인터랙티브 요소 추가 가능성이 있는 경우  
> 상대경로 완벽 처리, Azure/AKS 모두 최적, 확장성 최고
