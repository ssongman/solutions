# 관리 그룹 구조

관리 그룹(Management Group)은 **여러 구독을 계층적으로 묶어 정책과 RBAC을 일괄 적용**하는 Azure 기능입니다.  
랜딩존의 거버넌스 기반이 되는 핵심 구성 요소입니다.

---

## 권장 계층 구조

```
Root Management Group (Tenant Root)
│
├── Platform                          ← 공유 인프라 구독
│   ├── Identity                      (Azure AD DS, ADDS)
│   ├── Management                    (Log Analytics, Automation)
│   └── Connectivity                  (Hub VNet, ExpressRoute, DNS)
│
├── Landing Zones                     ← 워크로드 구독
│   ├── Corp                          (사내망 연결 필요 앱)
│   └── Online                        (인터넷 facing 앱)
│
├── Sandbox                           ← 개발/실험용 (정책 완화)
│
└── Decommissioned                    ← 폐기 예정 구독
```

---

## 관리 그룹별 역할

### Root Management Group
- 모든 구독에 공통 적용할 최상위 정책 배치
- **절대 삭제 불가** (테넌트 고정)
- 전사 보안 정책 (허용 리전, 필수 태그 등)

### Platform
| 하위 그룹 | 목적 | 주요 구독 리소스 |
|----------|------|----------------|
| Identity | AD 서비스 | Domain Controller VM, Azure AD DS |
| Management | 중앙 운영 | Log Analytics, Automation Account |
| Connectivity | 네트워크 허브 | Hub VNet, Firewall, ExpressRoute GW |

> Connectivity 구성 세부 내용 → [허브-스포크 토폴로지](../03.네트워킹/01.허브스포크.md)

### Landing Zones
| 하위 그룹 | 특징 |
|----------|------|
| Corp | Hub VNet에 Peering, 사내망 접근 가능 |
| Online | 직접 인터넷 노출 허용, DMZ 구성 |

### Sandbox
- 개발자가 자유롭게 실험하는 환경
- 비용 한도 정책 적용
- 운영 네트워크와 완전 격리

---

## 정책 상속 원리

```
Root MG 정책 → 모든 하위 구독에 자동 적용
     │
     └── Platform MG 정책 → Platform 구독에만 추가 적용
              │
              └── Connectivity MG 정책 → Connectivity 구독에만 추가 적용
```

- 상위 정책은 하위에서 **Override 불가** (Deny 정책)
- Allow 정책은 하위에서 **추가 허용** 가능

---

## 구독 수 가이드라인

| 환경 | 구독 분리 기준 |
|------|-------------|
| 소규모 (< 5팀) | 환경별 분리 (Dev / Prod) |
| 중규모 (5~20팀) | 팀 + 환경별 분리 |
| 대규모 (> 20팀) | 팀 + 환경 + 리전별 분리 |
