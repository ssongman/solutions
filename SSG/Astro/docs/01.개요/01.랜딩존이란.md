# 랜딩존이란?

Azure 랜딩존은 **엔터프라이즈 규모의 Azure 환경을 안전하고 일관성 있게 운영**하기 위한 아키텍처 패턴입니다.  
단순한 인프라 구성을 넘어 거버넌스, 보안, 네트워크, 운영 모델을 포함한 **전체 클라우드 운영 기반**을 정의합니다.

---

## 핵심 개념

### 왜 랜딩존이 필요한가?

클라우드 환경을 구성할 때 팀마다 다른 방식으로 리소스를 배포하면:

- 보안 정책 일관성 없음
- 비용 관리 어려움
- 네트워크 연결 복잡도 증가
- 컴플라이언스 대응 지연

랜딩존은 이러한 문제를 사전에 방지하는 **"클라우드 기반 공사"** 에 해당합니다.

---

## 구성 요소

```
Azure 랜딩존
├── 관리 그룹 (Management Group)   → 정책/권한 계층 관리
├── 구독 (Subscription)            → 리소스 격리 단위
├── 정책 (Azure Policy)            → 자동 거버넌스
├── RBAC                           → 최소 권한 접근
├── 네트워크 (Hub-Spoke)           → 중앙 집중 네트워크
├── 모니터링 (Log Analytics)       → 중앙 로그 수집
└── 보안 (Defender for Cloud)      → 통합 보안 관리
```

---

## 랜딩존 유형

| 유형 | 설명 | 대상 |
|------|------|------|
| **Platform Landing Zone** | 공유 서비스 (ID, 네트워크, 관리) | 플랫폼 팀 |
| **Application Landing Zone** | 실제 워크로드 배포 환경 | 개발/앱 팀 |

---

## Microsoft CAF와의 관계

Azure 랜딩존은 **Microsoft Cloud Adoption Framework (CAF)** 의 핵심 구성 요소입니다.

```
CAF 단계
Strategy → Plan → Ready(랜딩존) → Adopt → Govern → Manage
                      ↑
                  여기가 랜딩존
```
