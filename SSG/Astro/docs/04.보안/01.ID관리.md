# ID 관리 (Identity & Access Management)

Azure 랜딩존에서 ID 관리는 **최소 권한 원칙**을 기반으로  
Azure AD(Entra ID)와 RBAC을 통해 모든 접근을 제어합니다.

---

## 핵심 구성 요소

```
Azure AD (Entra ID) 테넌트
├── 사용자 / 그룹 / 서비스 주체
├── 조건부 액세스 정책
├── PIM (Privileged Identity Management)
└── MFA 정책
```

---

## RBAC 설계 원칙

### 1. 그룹 기반 RBAC (직접 사용자 할당 금지)

```
❌ 나쁜 예: user@company.com → Contributor 역할 직접 할당
✅ 좋은 예: AAD Group "lz-corp-contributors" → Contributor 역할 할당
                 └── user@company.com 이 그룹 멤버
```

### 2. 관리 그룹 레벨 역할 할당

| 역할 | 할당 범위 | 대상 그룹 |
|------|----------|----------|
| Owner | 구독 | 플랫폼 팀 리드 |
| Contributor | 리소스 그룹 | 애플리케이션 팀 |
| Reader | 관리 그룹 | 감사 팀 |
| Network Contributor | Connectivity 구독 | 네트워크 팀 |

### 3. 커스텀 역할 활용

```json
{
  "Name": "LZ-NetOps",
  "Actions": [
    "Microsoft.Network/virtualNetworks/read",
    "Microsoft.Network/virtualNetworks/subnets/*",
    "Microsoft.Network/routeTables/*"
  ],
  "NotActions": [
    "Microsoft.Network/virtualNetworks/delete"
  ],
  "AssignableScopes": [
    "/providers/Microsoft.Management/managementGroups/connectivity"
  ]
}
```

## PIM (Privileged Identity Management)

Owner, Contributor 등 **특권 역할은 PIM을 통해 Just-in-Time 방식**으로 활성화합니다.

```
평소: 권한 없음
    │
    │ 작업 필요 시 PIM 활성화 요청
    │
승인자 승인 (또는 자동 승인)
    │
최대 8시간 동안 권한 활성화
    │
작업 완료 → 자동 만료
```

```bash
# PIM 역할 활성화 (사용자가 직접 실행)
az rest --method POST \
  --url "https://management.azure.com/providers/Microsoft.Authorization/roleAssignmentScheduleRequests" \
  --body '{"properties": {"roleDefinitionId": "...", "justification": "긴급 배포 작업"}}'
```

## 조건부 액세스 정책

| 정책 | 조건 | 제어 |
|------|------|------|
| MFA 강제 | Azure Portal 접속 | MFA 필수 |
| 관리자 MFA | 권한 있는 역할 | MFA + 준수 디바이스 |
| 해외 IP 차단 | 비허용 국가 IP | 차단 |
| 레거시 인증 차단 | 레거시 프로토콜 | 차단 |

## 서비스 주체 & Managed Identity

애플리케이션이 Azure 리소스에 접근할 때는 **Managed Identity**를 우선 사용합니다.

```
❌ 나쁜 예: 서비스 주체 + 클라이언트 시크릿 (키 관리 필요)
✅ 좋은 예: System-assigned Managed Identity (Azure가 자동 관리)
```

```bash
# VM에 Managed Identity 활성화
az vm identity assign \
  --resource-group rg-corp \
  --name vm-app

# Key Vault 접근 권한 부여
az keyvault set-policy \
  --name kv-corp \
  --object-id <managed-identity-object-id> \
  --secret-permissions get list
```
