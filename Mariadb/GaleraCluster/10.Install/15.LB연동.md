# 15. KT Cloud LB

## 구성 목표

```
Client
  │
  ▼
[KT Cloud LB]  ← TCP 3306 (Public IP)
  │
  ├── maria01 (172.27.0.24:3306)
  ├── maria02 (172.27.0.53:3306)
  └── maria03 (172.27.0.175:3306)
```

- Galera Multi-Master 구성 → 모든 노드 Read/Write 가능
- LB를 단일 접속 엔드포인트로 사용
- 노드 장애 시 자동 제외 (Health Check)



---



# 1. KT Cloud 콘솔 - Load Balancer 생성

## 1) 콘솔 접속

KT Cloud 콘솔 → **Compute** → **Load Balancer** → **로드밸런서 생성**

## 2) 기본 정보 설정

| 항목 | 값 |
|------|-----|
| 이름 | maria-galera-lb |
| 네트워크 티어 | 해당 VM이 속한 티어 선택  KOR-Central A |
| 로드밸런싱 방식 | Round Robin |



## 3) 리스너(Listener) 설정

| 항목 | 값 |
|------|-----|
| 프로토콜 | TCP |
| 로드밸런서 포트 | 3306 |
| 인스턴스 포트 | 3306 |

> MariaDB는 TCP 프로토콜을 사용합니다. HTTP/HTTPS가 아닌 **TCP** 를 선택해야 합니다.



## 4) Health Check 설정

| 항목 | 값 |
|------|-----|
| Health Check 방식 | TCP |
| Health Check 포트 | 3306 |
| Ping 간격 | 10 (초) |
| Healthy Threshold | 2 |
| Unhealthy Threshold | 3 |

> TCP Health Check는 3306 포트 연결 가능 여부로 노드 상태를 판단합니다.



## 5) 인스턴스(Member) 등록   <-- 이 설정이 없음

**멤버 추가** 에서 Galera 노드 3개를 등록합니다.

| 인스턴스 | IP | 포트 |
|---------|-----|------|
| maria01 | 172.27.0.24 | 3306 |
| maria02 | 172.27.0.53 | 3306 |
| maria03 | 172.27.0.175 | 3306 |

**생성 완료** 클릭

# 3. 공인 IP 연결 (Static NAT / 포트포워딩)

KT Cloud는 LB 생성 후 공인 IP를 별도로 연결해야 외부 접속이 가능합니다.

## 1) 공인 IP 확인 또는 신청

**Network** → **IP 관리** → 사용 가능한 공인 IP 확인

공인 IP가 없으면 **공인 IP 신청** 으로 새로 할당받습니다.

## 2) LB에 공인 IP 연결

**Load Balancer** 목록에서 생성한 LB 선택 → **IP 연결** → 공인 IP 선택 후 적용

연결 완료 후 LB 상세 화면에서 공인 IP 확인:

```
공인 IP: 14.63.xxx.xxx
포트:    3306
```



---



# 4. 방화벽(ACL) 설정

KT Cloud 콘솔 → **Network** → **방화벽** 또는 **ACL 설정**

외부에서 LB 공인 IP의 3306 포트로 접근이 허용되도록 인바운드 규칙 추가:

| 방향 | 프로토콜 | 출발지 IP | 포트 |
|------|---------|----------|------|
| Inbound | TCP | 0.0.0.0/0 (또는 허용할 IP 대역) | 3306 |

> 보안을 위해 접속 허용 IP를 최소화하는 것을 권장합니다.



---



# 5. VM 방화벽 확인 (3노드 모두)

각 MariaDB 노드의 OS 방화벽에서 3306 포트가 열려 있는지 확인합니다.

```sh
firewall-cmd --list-ports
```

3306/tcp 가 없으면 추가:

```sh
firewall-cmd --permanent --add-port=3306/tcp
firewall-cmd --reload
```

# 6. 연결 테스트

## 1) LB를 통한 MariaDB 접속 확인

외부 클라이언트 또는 로컬 PC에서:

```sh
mysql -u appuser -p -h 14.63.xxx.xxx -P 3306

# 또는
mariadb -u appuser -p -h 14.63.xxx.xxx -P 3306
```

## 2) 접속 후 상태 확인

```sql
-- 현재 접속된 노드 확인
SELECT @@hostname;

-- Galera 클러스터 상태 확인
SHOW STATUS LIKE 'wsrep_cluster_size';
SHOW STATUS LIKE 'wsrep_node_name';
SHOW STATUS LIKE 'wsrep_connected';
SHOW STATUS LIKE 'wsrep_ready';
```

예상 결과:

```
+--------------------+-------+
| Variable_name      | Value |
+--------------------+-------+
| wsrep_cluster_size | 3     |
+--------------------+-------+

+--------------------+-------+
| Variable_name      | Value |
+--------------------+-------+
| wsrep_ready        | ON    |
+--------------------+-------+
```

## 3) LB 라운드로빈 확인

여러 번 접속해서 각기 다른 노드로 분산되는지 확인:

```sh
for i in {1..6}; do
  mysql -u appuser -pAppPass1234! -h 14.63.xxx.xxx -P 3306 -e "SELECT @@hostname;" 2>/dev/null
done
```

예상 출력:

```
+----------+
| maria01  |
+----------+
+----------+
| maria02  |
+----------+
+----------+
| maria03  |
+----------+
...
```

# 7. Failover 테스트

노드 1개를 내렸을 때 LB가 자동으로 제외하는지 확인합니다.

```sh
# maria02 에서 MariaDB 중지
systemctl stop mariadb
```

LB Health Check가 Unhealthy Threshold(3회) 실패 후 자동으로 maria02를 제외합니다.

클라이언트에서 재접속 시 maria01 또는 maria03 으로만 연결되는지 확인:

```sh
for i in {1..6}; do
  mysql -u appuser -pAppPass1234! -h 14.63.xxx.xxx -P 3306 -e "SELECT @@hostname;" 2>/dev/null
done
```

maria02 복구:

```sh
systemctl start mariadb
```

Health Check 통과 후 자동으로 다시 LB 멤버에 포함됩니다.

# 9. Trouble Shooting

## 1) LB에서 접속이 안 될 때 체크리스트

```
1. KT Cloud 방화벽(ACL)에서 3306 허용 확인
2. VM OS 방화벽에서 3306 열려있는지 확인 (firewall-cmd --list-ports)
3. MariaDB bind-address=0.0.0.0 설정 확인
4. 원격 접속 계정(appuser@'%') 생성 여부 확인
5. LB Health Check 상태 - 콘솔에서 멤버 상태가 Active인지 확인
```

## 2) bind-address 확인

```sh
grep -r 'bind-address' /etc/my.cnf /etc/my.cnf.d/
```

`bind-address=127.0.0.1` 이면 외부 접속 불가 → `0.0.0.0` 으로 변경 후 재시작

```sh
systemctl restart mariadb
```

## 3) LB Health Check 실패 원인

```sh
# MariaDB 서비스 상태 확인
systemctl status mariadb

# 3306 포트 리스닝 확인
ss -tlnp | grep 3306
```

## 4) 계정 권한 확인

```sql
SHOW GRANTS FOR 'appuser'@'%';
SELECT user, host FROM mysql.user;
```
