# 21. MariaDB Client

로컬 PC에서 Galera Cluster에 접속하기 위한 MariaDB Client 컨테이너 설정



---



# 1. 컨테이너 실행

```sh
docker run -d \
  --name mariadb-client \
  --restart unless-stopped \
  -e MARIADB_ALLOW_EMPTY_ROOT_PASSWORD=yes \
  mariadb:10.6 \
  sleep 365d
```

상태 확인:

```sh
docker ps | grep mariadb-client
```

예상 출력:

```
CONTAINER ID   IMAGE          COMMAND      STATUS         NAMES
xxxxxxxxxxxx   mariadb:10.6   "sleep 365d" Up 3 seconds   mariadb-client
```



---



# 2. 접속 정보 관리 (.env)

패스워드를 `.env` 파일에 저장하고 git에는 올리지 않습니다.

```sh
# .env 파일 위치: Mariadb/GaleraCluster/.env
MARIA_HOST=14.63.223.198
MARIA_PORT=3306
MARIA_USER=appuser
MARIA_PASSWORD=AppPass1234!
```

`.env`를 읽어서 접속:

```sh
source ../GaleraCluster/.env

docker exec mariadb-client \
  mariadb -u $MARIA_USER -p$MARIA_PASSWORD -h $MARIA_HOST -P $MARIA_PORT
```

> `.env` 는 `.gitignore`에 등록되어 있어 git에 커밋되지 않습니다.

# 3. Galera Cluster 접속


## 1) LB를 통한 접속 (권장)

```sh

docker exec -it mariadb-client bash

$ mariadb -u appuser -p -h 14.63.223.198 -P 3306

AppPass1234!


docker exec -it mariadb-client \
  mariadb -u appuser -p -h 14.63.xxx.xxx -P 3306
  
```

## 2) 특정 노드 직접 접속

```sh
# node1
docker exec -it mariadb-client \
  mariadb -u appuser -p -h 172.27.0.24 -P 3306

# node2
docker exec -it mariadb-client \
  mariadb -u appuser -p -h 172.27.0.53 -P 3306

# node3
docker exec -it mariadb-client \
  mariadb -u appuser -p -h 172.27.0.175 -P 3306
```

## 3) 컨테이너 쉘로 진입 후 접속

```sh
docker exec -it mariadb-client bash

# 컨테이너 내부에서
mariadb -u appuser -p -h 14.63.xxx.xxx -P 3306
```



---



# 3. 접속 확인

```sql
-- 접속된 노드 확인
SELECT @@hostname;

-- Galera 클러스터 상태 확인
SHOW STATUS LIKE 'wsrep_cluster_size';
SHOW STATUS LIKE 'wsrep_ready';

-- DB 목록
SHOW DATABASES;
```

# 4. 컨테이너 관리

```sh
# 중지
docker stop mariadb-client

# 삭제
docker rm mariadb-client

# 재시작
docker restart mariadb-client

# 로그 확인
docker logs mariadb-client
```
