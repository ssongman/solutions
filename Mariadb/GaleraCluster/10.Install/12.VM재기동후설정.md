

# 12. 재부팅 후 복구 절차



Galera는 전체 클러스터가 내려갔다가 다시 올라올 때, 보통 **가장 최신 상태의 노드 1대를 먼저 bootstrap**해서 Primary Component를 만든 뒤 나머지 노드를 붙여야 합니다. 또 현재 상태에 disabled로 보이므로 **부팅 시 자동 시작도 아직 활성화되지 않은 상태**입니다. 



## **1) 3대 모두에서 일단 상태 초기화**



각 노드에서 먼저:

```
systemctl stop mariadb
systemctl reset-failed mariadb
```

실패 원인 로그도 같이 확인해두세요.

```
journalctl -u mariadb -n 100 --no-pager
```

MariaDB는 시작 실패 원인이 거의 항상 서비스 로그나 에러 로그에 남는다고 안내합니다. 



------





## **2) 어떤 노드를 먼저 올릴지 확인**





각 노드에서 아래 파일을 확인하세요.

```
cat /var/lib/mysql/grastate.dat
```

여기서 중요한 값은:



- seqno
- safe_to_bootstrap





전체 클러스터가 모두 내려간 뒤에는 **가장 최신 데이터가 있는 노드**를 먼저 살려야 하고, 보통 safe_to_bootstrap: 1인 노드가 우선 후보입니다. MariaDB 문서도 전체 종료 후 복구 시 이런 상태 정보를 기준으로 bootstrap 대상을 고르도록 설명합니다. 



예시:

```
# GALERA saved state
version: 2.1
uuid: ...
seqno: 12345
safe_to_bootstrap: 1
```



------





## **3) 선택한 1대만 먼저 bootstrap**





예를 들어 song-maria01이 bootstrap 대상이면, 그 노드에서만:

```
systemctl set-environment _WSREP_NEW_CLUSTER='--wsrep-new-cluster'
systemctl start mariadb
systemctl unset-environment _WSREP_NEW_CLUSTER
```

MariaDB의 systemd 서비스는 _WSREP_NEW_CLUSTER 환경변수를 이용해 첫 노드를 새 클러스터로 시작할 수 있습니다. 



정상 확인:

```
systemctl status mariadb --no-pager
mariadb -uroot -e "SHOW STATUS LIKE 'wsrep_cluster_size';"
mariadb -uroot -e "SHOW STATUS LIKE 'wsrep_cluster_status';"
mariadb -uroot -e "SHOW STATUS LIKE 'wsrep_ready';"
```

정상 기대값:



- wsrep_cluster_size = 1
- wsrep_cluster_status = Primary
- wsrep_ready = ON 





------





## **4) 나머지 2대는 일반 시작**





첫 노드가 정상으로 올라온 뒤, 나머지 두 노드에서:

```
systemctl start mariadb
```

그 후 아무 노드에서:

```
mariadb -uroot -e "SHOW STATUS LIKE 'wsrep_cluster_size';"
```

정상이라면 다시 3이 나와야 합니다. Galera는 첫 노드가 Primary가 되면 나머지 노드가 그 클러스터에 join합니다. 



------





## **5) 자동 시작 설정**





현재 출력에 disabled라고 나왔으니, 3대 모두에서 아래를 실행해야 재부팅 후 자동으로 시작됩니다.

```
systemctl enable mariadb
```

MariaDB 공식 문서도 systemd 환경에서 부팅 시 자동 시작은 systemctl enable mariadb로 설정한다고 안내합니다. 



------





## **6) 지금 바로 필요한 최소 명령만 정리**







### **3대 모두**



```
systemctl stop mariadb
systemctl reset-failed mariadb
cat /var/lib/mysql/grastate.dat
```



### **bootstrap 대상 1대에서만**



```
systemctl set-environment _WSREP_NEW_CLUSTER='--wsrep-new-cluster'
systemctl start mariadb
systemctl unset-environment _WSREP_NEW_CLUSTER
mariadb -uroot -e "SHOW STATUS LIKE 'wsrep_cluster_size';"
mariadb -uroot -e "SHOW STATUS LIKE 'wsrep_cluster_status';"
```



### **나머지 2대**



```
systemctl start mariadb
```



### **마지막**



```
systemctl enable mariadb
```

지금 가장 빠른 방법은 **세 노드의 /var/lib/mysql/grastate.dat 내용**을 각각 확인하는 것입니다. 그걸 보면 어느 노드를 먼저 bootstrap해야 하는지 바로 정할 수 있습니다.
