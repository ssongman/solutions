# MariaDB Galera Cluster 부하 테스트



---



# 1. 구성 개요

```
로컬 PC
  │
  ├── load_test.py  ← Python 부하 테스트 스크립트
  │                    (INSERT / SELECT / UPDATE / DELETE 동시 실행)
  │
  ▼
[KT Cloud LB]  14.63.223.198:3306
  │
  ├── maria01 (172.27.0.24)
  ├── maria02 (172.27.0.53)
  └── maria03 (172.27.0.175)
```

**측정 항목**

| 항목 | 설명 |
|------|------|
| QPS | 초당 쿼리 수 |
| TPS | 초당 트랜잭션 수 (INSERT 기준) |
| Latency | Avg / Min / Max / P50 / P95 / P99 (ms) |
| 노드 분산 | LB를 통한 노드별 쿼리 분산 비율 |
| 에러율 | 오퍼레이션별 에러 수 |

# 2. 사전 준비

## 1) 패키지 설치

```sh
cd Mariadb/GaleraCluster/20.부하테스트

pip install -r requirements.txt
```

## 2) .env 확인

```sh
cat ../GaleraCluster/.env
```

```
MARIA_HOST=14.63.223.198
MARIA_PORT=3306
MARIA_USER=appuser
MARIA_PASSWORD=AppPass1234!
```

> `MARIA_DB` 를 지정하지 않으면 기본값 `loadtest` DB를 자동 생성합니다.



---



# 3. 부하 테스트 실행

## 기본 실행 (10 스레드 / 30초)

```sh
python load_test.py
```

## 옵션 지정

```sh
# 30 스레드 / 60초
python load_test.py --workers 30 --duration 60

# 50 스레드 / 120초 + 테스트 후 DB 삭제
python load_test.py --workers 50 --duration 120 --drop

# .env 경로 직접 지정
python load_test.py --workers 20 --duration 60 --env /path/to/.env
```

## 옵션 설명

| 옵션 | 기본값 | 설명 |
|------|-------|------|
| `--workers` | 10 | 동시 접속 스레드 수 |
| `--duration` | 30 | 테스트 지속 시간 (초) |
| `--env` | `../.env` | .env 파일 경로 |
| `--drop` | false | 테스트 후 loadtest DB 삭제 |

# 5. HTML 리포트

테스트 완료 후 자동으로 HTML 리포트가 생성됩니다.

```
20.부하테스트/
  └── report_20260407_153012.html   ← 브라우저로 열기
```

**리포트 포함 항목**

- 요약 카드 (총 쿼리 / QPS / TPS / 동시 접속 / 에러 수)
- 오퍼레이션별 레이턴시 테이블 (Avg / Min / Max / P50 / P95 / P99)
- 노드별 쿼리 분산 (바 차트)
- 에러 현황
- 테스트 설정 정보

# 6. 단계별 테스트 시나리오 (권장)

## Step 1. 워밍업 (기준 성능 측정)

```sh
python load_test.py --workers 5 --duration 30
```

## Step 2. 일반 부하

```sh
python load_test.py --workers 20 --duration 60
```

## Step 3. 고부하

```sh
python load_test.py --workers 50 --duration 60
```

## Step 4. 한계 테스트

```sh
python load_test.py --workers 100 --duration 60
```

각 단계 리포트를 비교하여 **TPS 포화 지점**과 **P99 레이턴시 급등 구간**을 확인합니다.

# 7. Failover 테스트 (선택)

부하 테스트 중 노드 1개를 내려서 자동 페일오버 확인

**터미널 1** - 부하 테스트 실행

```sh
python load_test.py --workers 20 --duration 120
```

**터미널 2** - 노드 중단 (maria02에서)

```sh
ssh root@172.27.0.53 "systemctl stop mariadb"
```

리포트에서 해당 구간의 에러율과 노드 분산 변화를 확인합니다.

노드 복구:

```sh
ssh root@172.27.0.53 "systemctl start mariadb"
```
