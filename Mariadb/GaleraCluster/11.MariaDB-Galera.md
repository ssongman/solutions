

# 1. node1 설치



### 1) 모듈 상태 확인

```
dnf module list mariadb
dnf module reset mariadb -y
dnf module disable mariadb -y
```

여기서 mariadb 모듈이 enabled로 남아 있으면 안 됩니다. RHEL 8 계열은 AppStream 모듈 구조를 사용하므로, 이 단계가 중요합니다. 



### 2) MariaDB.repo

/etc/yum.repos.d/MariaDB.repo를 아래처럼 수정

```
[mariadb]
name = MariaDB
baseurl = https://rpm.mariadb.org/10.6/rhel8-amd64
gpgkey = https://rpm.mariadb.org/RPM-GPG-KEY-MariaDB
gpgcheck = 1
module_hotfixes = true
```

이 옵션은 DNF 문서에서 **모듈러 필터링을 적용하지 않도록 하는 방법**으로 설명됩니다. 





### **3) 캐시 완전 정리**

```
dnf clean all
rm -rf /var/cache/dnf
dnf makecache
```

MariaDB 문서도 repo 변경 후 dnf clean all로 메타데이터를 정리하는 흐름을 안내합니다. 



### **4) 이제 패키지가 보이는지 확인**

```
dnf list --showduplicates MariaDB-server
dnf search MariaDB-server
```

정상이라면 MariaDB-server.x86_64 ... mariadb-main 같은 항목이 보여야 합니다. MariaDB 문서 예시에서도 공식 repo 패키지와 AppStream의 mariadb-server가 함께 보일 수 있다고 설명합니다. 





### **5) 설치**

```sh

$ dnf install -y MariaDB-server MariaDB-client galera-4


# 확인
rpm -ql MariaDB-server | egrep 'mariadb.service|/usr/sbin/mariadbd'
ls -l /usr/sbin/mariadbd
systemctl list-unit-files | grep -i maria


	•	/usr/sbin/mariadbd
	•	mariadb.service
	


# 참고
$ dnf install -y MariaDB-server MariaDB-client galera-4 MariaDB-shared MariaDB-common


```

MariaDB 공식 문서도 Galera 구성 시 MariaDB-server MariaDB-client galera-4 설치를 안내합니다. 



```sh


# 삭제시...
$ dnf remove -y MariaDB-server MariaDB-client MariaDB-common MariaDB-shared galera-4 mariadb\* galera\*

$ dnf autoremove -y


# 데이터까지 완전삭제
rm -rf /var/lib/mysql
rm -rf /etc/my.cnf.d

# 확인
$ systemctl status mariadb



```







### **6) 설치 후 확인**



```
which mariadbd
which galera_new_cluster
rpm -qa | egrep 'MariaDB|galera'
```

정상이면 보통 아래처럼 나와야 합니다.

```
/usr/sbin/mariadbd
/usr/bin/galera_new_cluster
```

# 4. Galera 설정 (핵심 ⭐)

 3노드 모두 동일 파일 생성

```
vi /etc/my.cnf.d/galera.cnf
```



## node1 (maria01)



```
[mariadb]
bind-address=0.0.0.0

binlog_format=ROW
default_storage_engine=InnoDB
innodb_autoinc_lock_mode=2

wsrep_on=ON
wsrep_provider=/usr/lib64/galera-4/libgalera_smm.so

wsrep_cluster_name="galera_cluster"
wsrep_cluster_address="gcomm://172.27.0.24,172.27.0.53,172.27.0.175"

wsrep_node_name="maria01"
wsrep_node_address="172.27.0.24"

wsrep_sst_method=rsync
```



## node2 (maria02)



```
wsrep_node_name="maria02"
wsrep_node_address="172.27.0.53"
```



## node3 (maria03)



```
wsrep_node_name="maria03"
wsrep_node_address="172.27.0.175"
```

# 10. Troubld Shooting



## 1) 가장 많이 터지는 문제



👉 node2에서 join 실패

```
journalctl -u mariadb -n 50
```

보통 원인:



- 포트 막힘 ❌
- IP 오타 ❌
- wsrep 설정 틀림 ❌
- mysql 데이터 남아있음 ❌



## 2) cluster 안 붙는 경우



```
SHOW STATUS LIKE 'wsrep_cluster_status';
```

👉 정상:

```
Primary
```



## 3) stuck 되는 경우

👉 무조건 초기화

```
rm -rf /var/lib/mysql/*
```
